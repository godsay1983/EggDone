# EggDone 功能优化 TODO

本文档规划 EggDone 面向个人使用的后续功能。目标是提升日常记录、查找和按时完成任务的效率，同时保持托盘工具轻量、操作直接、离线可用。

工程发布、代码签名和跨平台实机验证继续由 [OPTIMIZATION_TODO.md](OPTIMIZATION_TODO.md) 跟踪。

## 产品原则

- 默认面板保持简洁，新增字段按需展开。
- 常用操作最多经过一次点击或一次快捷键。
- 新功能必须兼容 SQLite、JSON 导入导出和 S3 / MinIO 同步。
- 不因提醒、分组或重复任务阻塞基础 Todo 操作。
- Windows 优先实现，数据模型保持跨平台。
- 先完成单层结构，不演变为完整项目管理系统。

## F1：查找与列表整理

本阶段成本低、风险小，任务数量增加后收益明显。

### 搜索

- [x] 在面板顶部增加可收起的搜索框。
- [x] 按任务标题进行即时、不区分大小写的本地过滤。
- [x] 搜索只影响当前显示结果，不修改任务顺序。
- [x] 支持 `Escape` 清空搜索并返回新增输入框。
- [x] 无匹配结果时显示明确空状态。

### 已完成任务显示

- [x] 增加“显示 / 隐藏已完成”切换。
- [x] 默认记住用户上次选择。
- [x] 隐藏已完成时，未完成统计保持准确。
- [x] 搜索命中已完成任务时遵守当前显示设置。

### 置顶

- [x] 为 Todo 增加可选置顶状态。
- [x] 置顶任务位于当前列表顶部，组内仍支持拖动排序。
- [x] 置顶状态参与 JSON 导入导出和云同步。
- [x] 限制置顶入口的视觉权重，避免每行按钮过多。

验收标准：

- 500 条任务下输入搜索词时无明显卡顿。
- 搜索、完成状态筛选、置顶和拖动排序可以组合使用。
- 重启后显示设置、置顶状态和顺序保持不变。

实现说明：搜索结果是任务列表的临时子集。输入搜索词时禁用拖动和上下移动，清空搜索后恢复，避免局部结果改变完整列表顺序。

置顶顺序按“未完成置顶、未完成普通、已完成置顶、已完成普通”排列，各组内部保留独立拖动顺序。

## F2：日期、提醒与今天视图

这是下一阶段最有价值的核心功能。

### 数据模型

- [x] 为 Todo 增加可选到期信息，区分“纯日期”和“具体时间”。
- [x] 纯日期使用本地日历日期语义，具体时间使用 `due_at` UTC 毫秒时间戳。
- [x] 增加可选 `reminder_at` 提醒时间。
- [x] 按 Todo UUID、设备和提醒时间记录本机已触发通知，防止重复提醒。
- [x] 本机通知记录不参与同步，避免一台设备提醒后抑制其他设备通知。
- [x] 具体时间按 UTC 毫秒时间戳存储，界面按本地时区显示。
- [x] 提供数据库 migration，并验证旧数据库升级。
- [x] 扩展 JSON 和同步格式，旧客户端忽略未知字段时不破坏数据。

### 日期设置

- [x] 在任务编辑区域提供轻量日期入口。
- [x] 提供“今天”“明天”“下周”和“自定义”快捷选项。
- [x] 支持只设置日期，不强制设置具体时间。
- [x] 支持清除到期时间和提醒时间。
- [x] 逾期任务使用克制但清晰的视觉提示。

### 系统提醒

- [ ] 接入 Tauri 系统通知能力。
- [ ] 首次使用提醒时请求通知权限。
- [ ] 到达提醒时间后发送系统通知。
- [ ] 应用启动或从休眠恢复后补发尚未处理的到期提醒。
- [ ] 完成、删除或取消提醒后不再通知。
- [ ] 点击通知时打开面板并定位对应任务。
- [ ] 提供“稍后 10 分钟”和“今天晚些时候”操作；平台不支持通知按钮时回退到面板操作。

### 今天视图

- [x] 增加“全部 / 今天”紧凑视图切换。
- [x] 今天视图包含今日到期和已经逾期的未完成任务。
- [x] 清楚区分“今天”和“逾期”。
- [x] 托盘菜单增加“今天任务”入口，但不扩大常驻菜单复杂度。
- [x] 托盘提示可显示今天剩余任务数。

验收标准：

- 提醒不会因重启、休眠恢复或时区显示转换而重复或遗漏。
- 没有通知权限时，本地 Todo 仍可正常使用并显示设置提示。
- 两台设备同步到期和提醒时间后，可分别触发本机通知。
- 今天视图不隐藏逾期任务。

## F3：单层分组

提醒与今天视图稳定后再加入分组，避免同时改变列表结构和时间模型。

### 分组模型

- [ ] 新建 `groups` 表，包含 UUID、名称、颜色、排序和软删除字段。
- [ ] Todo 增加可空的 `group_uuid`，未分组任务继续有效。
- [ ] 分组名称不能为空，同名是否允许在实现前确定。
- [ ] 分组和归属关系加入 migration、JSON 和同步格式。
- [ ] 删除分组时保留任务，并将其移动到“未分组”。

### 分组交互

- [ ] 面板顶部增加“全部 / 未分组 / 指定分组”筛选。
- [ ] 新增任务时默认使用当前分组。
- [ ] 支持在任务编辑中移动分组。
- [ ] 支持拖动任务到分组，但不要求首版实现跨分组拖动。
- [ ] 分组管理只支持新增、重命名、排序和删除。
- [ ] 限制颜色数量，使用现有主题的柔和预设色。

验收标准：

- 不使用分组时，面板操作复杂度与当前版本基本一致。
- 分组筛选、搜索、今天视图和已完成筛选可以组合。
- 分组删除、跨设备同步和 JSON 导入不会产生孤立任务。

## F4：重复任务与快捷新增

这两项依赖稳定的日期模型，应最后实现。

### 重复任务

- [ ] 首版只支持每天、每周、每月和工作日。
- [ ] 记录重复规则和下一次到期时间。
- [ ] 完成当前任务后生成下一实例，不提前批量创建。
- [ ] 编辑时区分“仅此任务”和“后续任务”。
- [ ] 删除时区分“仅此任务”和“整个重复规则”。
- [ ] 明确定义错过多次后的补建策略，默认只生成下一实例。
- [ ] 为跨月日期、夏令时和多设备同时完成增加测试。

### 快捷新增语法

- [ ] 支持识别“今天”“明天”“周五”等日期词。
- [ ] 支持识别明确时间，例如 `明天 10:00`。
- [ ] 分组稳定后再评估 `#工作` 语法。
- [ ] 解析结果在提交前提供可取消的视觉提示。
- [ ] 解析失败时保留完整标题，不阻止任务创建。
- [ ] 不引入大型自然语言处理依赖。

验收标准：

- 重复任务不会重复生成、漏生成或无限补建。
- 快捷语法不改变普通标题输入的可靠性。
- 无法识别的文字始终作为任务标题保存。

## 暂不计划

- 多级项目、子任务和依赖关系。
- 分组与多标签同时存在。
- 看板、甘特图和完整日历视图。
- 团队协作、任务指派和评论。
- 工时统计、复杂报表和积分系统。
- 富文本、附件和云文件管理。
- AI 自动拆分或自动安排任务。

## 推荐实施顺序

1. 搜索与已完成任务折叠。
2. 置顶任务。
3. 到期时间与今天视图。
4. 系统通知与稍后提醒。
5. 单层分组。
6. 重复任务。
7. 快捷新增语法。

每完成一个阶段，都应同步更新 README、数据库 migration、导入导出格式、同步文档和自动化测试。
