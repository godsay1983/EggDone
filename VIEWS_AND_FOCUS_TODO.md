# EggDone 视图与专注 TODO（F6–F8）

本文档规划 EggDone 桌面端的「优先级 + 四象限」「日历 / 日程视图」「番茄钟 / 专注」三组新功能。它是 [FUNCTION_OPTIMIZATION_TODO.md](FUNCTION_OPTIMIZATION_TODO.md) 中 F1–F5 的延续，单独成档以便聚焦实现。工程发布、签名和跨平台实机验证仍由 [OPTIMIZATION_TODO.md](OPTIMIZATION_TODO.md) 跟踪。

UI 设计预览见 `mockups/eggdone-features-mockup.html`。

## 产品原则

- 守住「轻量、离线、托盘失焦隐藏」定位，不演变为完整项目管理系统。
- **唯一数据本体是 Todo**。四象限、日历、全部、今天都是看同一批 Todo 的不同「视图」，不复制数据。
- 番茄钟是作用在 Todo 上的「工具」，不是视图，独立成悬浮窗，不塞进主面板的视图切换。
- 新功能必须兼容 SQLite、JSON 导入导出和 S3 / MinIO 同步；新字段对旧客户端「未知即忽略、缺失给默认值」。
- 视图切换不丢失当前分组筛选；常用操作最多一次点击或一次快捷键。
- Windows 优先实现，数据模型保持跨平台，与鸿蒙端 `priority` 语义一致。

## F6：任务优先级与四象限

四象限 = 重要程度（`priority`）× 紧急程度（由到期日推导）。先补「重要」维度，再做视图。

### 数据模型

- [x] 为 `todos` 增加 `priority` 字段：整数，`NOT NULL DEFAULT 0`，`0 = 普通`、`1 = 重要`（预留升级到多级）。
- [x] 提供顺序 migration（`db.rs`），验证旧库升级到默认 `0`。
- [x] `commands.rs` 读写 `priority`，设置 / 取消重要时更新 `updated_at` 和 `updated_by`。
- [x] `data_exchange.rs` 的 JSON 导入导出包含 `priority`。
- [x] `sync.rs` 序列化包含 `priority`；合并仍按整条记录比 `updated_at`，不新增合并规则。
- [x] `types.ts` 与 `todoStore.ts` 增加 `priority`，旧数据缺字段时回退 `0`。

### 紧急度推导（不新增字段）

- [x] 紧急 = 逾期或今天到期；无到期日 = 不紧急。复用现有「今天视图」语义。
- [x] 四象限映射：重要且紧急 / 重要不紧急 / 不重要紧急 / 不重要不紧急。

### 桌面 UI

- [x] 任务编辑 / 悬停菜单增加「设为重要 / 取消重要」，与「置顶」同级，视觉克制。
- [x] 重要任务显示一个小色点 / 旗标，不增加每行按钮密度。
- [x] 快捷新增支持开头 `!` / `！` 标记重要，预览与保存结果一致。
- [x] 视图切换器增加「四象限」（当前为 `全部 / 今天 / 四象限`，日历在 F7 实现）。
- [x] 四象限视图：顶部 2×2 概览（只显计数，点格子等价筛选）+ 下方按象限分区列表。
- [x] 空象限显示克制的拖拖蛋空状态文案。
- [x] 默认不改现有手动拖动顺序；优先级仅作为视图维度和可选排序。

**验收标准：**

- 不使用优先级时，列表行为与当前版本基本一致。
- 四象限与搜索、分组筛选、显示 / 隐藏已完成可组合。
- 设置 / 取消重要后跨设备同步正确，旧客户端读到 `priority` 不报错。
- 四象限象限归属随到期日变化自动更新。

## F7：日历 / 日程视图

不改数据模型，完全复用 `due_date` / `due_at`。桌面面板窄，做轻量「日程」为主，月历用迷你 popover 跳转。

### 日程（Agenda）

- [x] 视图切换器增加「日历」。
- [x] 按日期分桶：逾期 / 今天 / 明天 / 本周内 / 更晚 / 无到期日，每桶 section 标题与计数。
- [x] 遵守「显示 / 隐藏已完成」，可与分组筛选、搜索组合。
- [x] 分桶为纯前端逻辑（`todoStore.ts` + `TodoPanel.svelte`），无后端改动。

### 周条与迷你月历

- [x] 视图顶部增加「7 天周条」，今天高亮，有任务的天带计数。
- [x] 提供「跳转日期」入口选择日期，选完聚焦该天任务。
- [x] 桌面端不做常驻完整月历（面板太小，且在「暂不计划」内），保留周条 + 跳转日期。

**验收标准：**

- 500 条任务下切换到日程视图无明显卡顿。
- 分桶口径在跨时区显示下清晰，逾期不被隐藏。
- 周条与列表的「今天」锚点一致（蛋黄环）。

## F8：番茄钟 / 专注（独立悬浮窗）

桌面端不复用会自动隐藏的主面板，改用独立悬浮计时窗承载，避免破坏托盘失焦隐藏模型。v1 不落库、不做统计。

- [x] 前台专注 MVP：主面板入口 + 专注弹层，默认 25 分钟 / 休息 5 分钟，支持暂停 / 继续、+5 分钟、跳过、结束，不落库、不参与同步。
- [x] 独立专注窗口基础：新增隐藏的 `focus` Tauri 窗口、专用前台计时 UI，并由主面板「专注」入口打开。
- [x] 将前台 MVP 迁移为独立 Tauri 悬浮窗。

### 窗口与生命周期

- [x] 新建无边框、置顶、跳过任务栏的悬浮计时窗（独立 Tauri window）。
- [x] 悬浮窗失焦不隐藏，与 Todo 主面板生命周期分离。
- [ ] 支持展开态（含操作）与收起胶囊态，可拖拽到屏幕角落。
- [x] 计时基于系统时间戳，休眠 / 恢复后按真实时间校正，不靠累加。

### 计时与操作

- [x] 默认专注 25 分钟 / 休息 5 分钟，时长在设置中可配。
- [x] 核心操作仅四个：暂停 / 继续、+5 分钟、跳过、结束。
- [x] 可选关联一条 Todo（临时指针，仅展示，不落库）。
- [x] 可选「专注结束 → 标记该 Todo 完成」（动作触发，非新数据）。
- [x] 专注 / 休息阶段使用不同柔和配色，保持与移动端一致。

### 托盘与提醒

- [x] 托盘 tooltip 显示 `专注 mm:ss · 任务名`。
- [ ] 托盘右键增加 `开始专注 / 暂停 / 结束专注`，不开窗也能控制。
- [x] 阶段结束发 Windows 系统通知（复用现有 toast，可带「开始休息 / 再来一颗」按钮）。

**验收标准：**

- 关闭 / 隐藏 Todo 主面板后，悬浮计时窗仍正常走时。
- 系统休眠后恢复，剩余时间按真实时间正确。
- 退出应用不残留悬浮窗与托盘计时状态。

## 数据模型与同步影响汇总

| 功能 | 是否改库 | 同步影响 | 可分端先后上线 |
| --- | --- | --- | --- |
| F6 优先级 / 四象限 | 是（加 `priority`） | 随整条记录合并，无新规则 | 是（未知字段忽略 / 缺失给默认） |
| F7 日历 / 日程 | 否（复用 `due_date`） | 无 | 是 |
| F8 番茄钟 / 专注 | 否（v1 不落库） | 无 | 是 |

## 暂不计划

- 多级优先级权重排序与复杂自动排程。
- 番茄钟工时统计、历史报表、积分系统。
- 桌面端常驻完整月历视图。
- 四象限内任务跨格拖动（首版用菜单改重要 / 紧急即可）。

## 推荐实施顺序

1. F6 数据模型：`priority` 字段、migration、JSON / 同步兼容。
2. F6 桌面 UI：重要标记 + 四象限视图。
3. F7 日历 / 日程视图与周条。
4. F8 番茄钟悬浮窗 + 托盘集成 + 到点通知。

每完成一项，同步更新 README、数据库 migration、导入导出格式、同步文档与自动化测试。
