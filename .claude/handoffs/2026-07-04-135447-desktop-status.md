## Codebase Understanding

### Architecture Overview

EggDone desktop is a Svelte + Tauri tray app. The Todo panel is implemented mostly in `src/lib/components/TodoPanel.svelte`, individual rows in `src/lib/components/TodoItem.svelte`, and shared visual styling in `src/app.css`. Release metadata is split across frontend package metadata and Tauri/Rust metadata, so version bumps must be synchronized across `package.json`, `src-tauri/Cargo.toml`, `src-tauri/tauri.conf.json`, and the `eggdone` package entry in `src-tauri/Cargo.lock`.

### Key Patterns Discovered

- Keep visible version strings aligned with release metadata; the about dialog is not derived automatically from `package.json`.
- Task row metadata is compact and badge-based. New metadata should join the existing `.todo-meta` row instead of adding another action column.
- Group colors use `data-group-color` plus CSS variables. New group-aware UI should reuse the same token pattern instead of hardcoding each color locally.
- The app is intentionally lightweight. Avoid adding larger UI frameworks or new state systems for small presentation changes.
