## Architecture Overview

- Svelte components under `src/lib/components/` own interaction and presentation.
- Frontend date formatting and local datetime conversion live under `src/lib/utils/`.
- Tauri commands in `src-tauri/src/commands.rs` validate schedules and create future repeat instances.

- `src-tauri/src/sync.rs` and `src-tauri/src/data_exchange.rs` canonicalize timed repeating tasks to date-only transfer records, then restore local timestamps on persistence.

## Key Patterns Discovered

- A Todo must not persist both `due_date` and `due_at`.
- New desktop schedules use `due_at`; legacy `due_date` tasks remain readable and become `18:00` when edited.
- Repeating tasks use `due_at` locally but are serialized with `due_date` and `due_at: null`.
- When a synced repeating task is restored, an existing local hour/minute is preserved; a new device defaults to `18:00`.
- SQL uses bound parameters, and Rust business errors are returned as readable Chinese strings.
- Keep sync protocol version 1 compatible until both desktop and Harmony clients are deliberately migrated together.

## Decisions Made

| Decision | Options Considered | Rationale |
|----------|-------------------|-----------|
| Release as `0.1.1` | `0.1.1`, `0.2.0` | This is a backward-compatible feature enhancement before a stable release |
