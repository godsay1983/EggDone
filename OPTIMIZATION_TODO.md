# EggDone 优化 TODO

本文档用于跟踪 EggDone 从 MVP 到可稳定发布版本的优化工作。任务按依赖关系排列；除紧急缺陷外，不跨阶段开发。

## 目标

- 保持托盘 Todo 的轻量定位。
- 提升数据可靠性、任务操作效率和原生桌面体验。
- 支持 S3、MinIO 等 S3 兼容存储的跨设备同步。
- 建立可测试、可迁移、可发布的工程基础。

## P0：数据与运行基础

后续任务编辑、排序和同步都依赖本阶段。

### 数据库迁移

- [x] 建立 `schema_migrations` 表和顺序 migration 机制。
- [x] 为现有 Todo 生成全局唯一 `uuid`。
- [x] 新增 `completed_at`、`deleted_at`、`sort_order` 字段。
- [x] 将时间字段统一为 UTC 毫秒时间戳。
- [x] 保证旧数据库升级后数据不丢失。
- [x] 为 migration 增加全新数据库和旧数据库升级测试。

验收标准：

- 旧版 `eggdone.sqlite3` 可直接升级。
- 重复执行 migration 不会报错或重复修改数据。
- 新增、完成、排序和软删除字段均有明确默认值。

### 单实例运行

- [x] 接入 Tauri 单实例插件。
- [x] 重复启动时激活已有进程并显示 Todo 面板。
- [x] 将第二个进程收到的参数转发给主进程。

验收标准：

- 任意时刻只有一个 EggDone 进程。
- 再次启动不会创建第二个托盘图标。

### 数据备份与交换

- [x] 增加 JSON 导出。
- [x] 增加 JSON 导入预览和确认。
- [x] 导入时按 UUID 合并，不直接覆盖数据库。
- [x] 增加手动 SQLite 备份。
- [x] 导入、导出格式包含 `format_version`。

验收标准：

- 导出后可以在全新数据库中恢复全部任务。
- 无效或高版本 JSON 不会破坏本地数据。

## P3：S3 / MinIO 同步

第一版只实现手动同步。自动同步在冲突和数据安全验证通过后再启用。

### 同步数据模型

- [x] 定义版本化 `todos.json` 格式。
- [x] 同步内容包含 Todo UUID、时间、排序和软删除状态。
- [x] 生成并持久化本机 `device_id`。
- [x] 按 UUID 合并本地和远端任务。
- [x] 冲突默认采用较新 `updated_at`，相同时间按稳定规则决胜。
- [x] 为合并、删除和排序冲突编写单元测试。

建议对象结构：

```text
eggdone/
└─ users/
   └─ <sync-id>/
      └─ todos.json
```

### S3 / MinIO 配置

- [x] 支持自定义 Endpoint。
- [x] 支持 AWS S3、MinIO 和其他 S3 兼容服务。
- [x] 支持 HTTP 和 HTTPS Endpoint。
- [x] HTTP 配置显示明文传输警告，但允许用户继续。
- [x] 支持 Region、Bucket、Object Key 和 Path Style。
- [x] 增加“测试连接”。
- [x] 增加“立即同步”。

### 凭据安全

- [x] Access Key 和 Secret Key 保存到系统凭据库。
- [x] SQLite 只保存非敏感同步配置。
- [x] 日志和错误信息不得包含 Secret Key。
- [x] 提供删除凭据和禁用同步操作。

### 冲突保护

- [x] 下载对象时记录 ETag。
- [x] 上传时使用 `If-Match`。
- [x] 首次创建使用 `If-None-Match: *`。
- [x] ETag 冲突后重新下载、合并并重试一次。
- [x] 重试仍失败时停止写入并保留本地数据。
- [x] 显示同步中、已同步、离线、冲突和失败状态。

### 自动同步

- [x] 启动后拉取一次。
- [x] 本地修改后延迟 3 至 5 秒同步。
- [x] 同一时间只允许一个同步任务运行。
- [x] 网络失败使用有限次数指数退避。
- [x] 应用退出前不强制等待长期网络请求。

验收标准：

- 两台设备可新增、编辑、完成和删除任务，合并后不丢数据。
- 并发修改不会无条件覆盖远端新版本。
- MinIO HTTP、MinIO HTTPS 和至少一种标准 S3 服务通过验证。
- 断网和凭据错误不会影响本地 Todo 使用。

## 推荐版本划分

### v0.2：可靠数据与任务体验

- 数据库 migration
- UUID、完成时间、软删除和排序
- 单实例运行
- 任务编辑、清除已完成、拖动排序和删除撤销

- UUID 合并
- ETag 冲突保护

## 暂不计划

- 复杂分类、项目和标签系统
