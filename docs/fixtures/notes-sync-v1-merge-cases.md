# Note Sync V1 合并样例

两端实现必须对以下输入得出相同结果：

| 场景 | 本地记录 | 远端记录 | 结果 |
|---|---|---|---|
| 更新时间不同 | `updated_at = 1000` | `updated_at = 2000` | 远端胜出 |
| 同时删除 | 活动记录，`updated_at = 2000` | 墓碑，`updated_at = 2000` | 墓碑胜出 |
| 时间和删除状态相同 | `updated_by = 1111...` | `updated_by = 2222...` | UUID 字典序较大的设备胜出 |
| 文本不同但前述字段相同 | 标题或正文较小 | 标题或正文较大 | 按 `pinned, color, title, content, created_at` 顺序稳定决胜 |
| 只有一端存在 | 本地存在 | 远端缺失 | 保留已有记录 |

排序与决胜字段按实现方案执行，不能使用数组原始顺序作为结果依据。
