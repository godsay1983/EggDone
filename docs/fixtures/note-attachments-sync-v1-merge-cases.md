# Note Attachment Sync V1 合并样例

两端实现必须对以下输入得出相同结果：

| 场景 | 本地记录 | 远端记录 | 结果 |
| --- | --- | --- | --- |
| 更新时间不同 | `updated_at = 1000` | `updated_at = 2000` | 远端胜出 |
| 删除冲突 | 活动记录，`updated_at = 2000` | 墓碑，`updated_at = 2000` | 墓碑胜出 |
| 时间和删除状态相同 | `updated_by = 1111...` | `updated_by = 2222...` | UUID 字典序较大的设备胜出 |
| 元数据不同但前述字段相同 | 左侧规范字段较小 | 右侧规范字段较大 | 按实现方案的规范字段顺序稳定决胜 |
| 只有一端存在 | 本地存在 | 远端缺失 | 保留已有记录 |
| 父便签暂时不存在 | 附件 `note_uuid` 有效 | Note 尚未同步完成 | 保留元数据，暂不展示 |

数组输入顺序不得影响合并结果。二进制内容不可变，不参与 JSON 合并。
